# 顧客別納入分析システム 仕様書

---

## システム名称

- 日本語名：顧客別納入分析システム
- 配布 exe 名（PyInstaller 既定）：`顧客別納入分析システム.exe`
- リポジトリ／パッケージ名：`customer_delivery_analytics`

---

## 1. 概要

営業部向けに、Access の受注データ App を参照し、顧客別を主軸として納入実績を一覧確認できるデスクトップアプリを作成する。  
あわせて、過去実績をもとに将来の納入数・金額を年単位で予測できるようにする。

本アプリは Windows デスクトップアプリとし、Python で実装する。  
CSV への中間出力は行わず、Access から直接データ取得する。

---

## 2. データ参照元

`\\192.168.1.200\共有\生産管理課\受注データApp.accdb`  
（環境変数 `CDA_ACCESS_DB` で上書き可能。実装は `app/config/settings.py` の `ENV_ACCESS_DB`。）

---

## 3. 使用テーブル

| テーブル | 用途 |
|----------|------|
| `t_納品` | 納入明細の取得・品番一覧 |
| `t_製品マスタ` | 明細と結合し客先名（顧客表示）を付与。顧客名一覧（DISTINCT） |
| `t_客先マスタ` | 顧客コード・客先名の一覧（プルダウン・表示用） |

---

## 4. アプリが SQL で取得するカラム

**`t_納品` と `t_製品マスタ` を結合した明細クエリ**では、次の列のみを SELECT する（一覧・集計・予測で使用する最小限。転送・メモリ軽量化のため **`t_納品.単価` は取得しない**。DB 上に単価列があっても参照しない）。

| 論理名 | 元 |
|--------|-----|
| 納入日 | `t_納品.納入日` |
| 顧客 | `Trim(t_製品マスタ.客先名)` |
| 品番 | `Trim(t_納品.品番)` |
| 納品数 | `t_納品.納品数` |
| 金額 | `t_納品.金額` |

**`t_客先マスタ`**：顧客プルダウン用に `コード`・`客先`（いずれも Trim 前提）を DISTINCT 取得する。

**その他の参照クエリ**：品番 DISTINCT、顧客別品番一覧など、用途に応じた最小列のみ取得する（実装は `app/service/delivery_service.py`）。

---

## 5. 結合条件

`t_納品.品番` と `t_製品マスタ.製品番号` を `Trim` 済みで一致させて結合する。  
（※旧仕様の「製品名」は名称列のため誤り。実 DB は製品番号が品番コードに対応する。）

---

## 6. 機能概要

- 顧客別実績一覧（集計単位：顧客別／品番別／顧客×品番別）
- 品番別分析
- 年月集計
- 年単位予測（直線延長・重み付き回帰・外部要因予測。数値計算は **numpy の加重最小二乗**（`numpy.linalg.lstsq`）。scikit-learn は使用しない）
- Excel 出力
- 年別・月別グラフ（matplotlib）

---

## 7. 非機能要件

- 起動時全件ロード禁止
- SQL で絞り込み
- 10 万件対応
